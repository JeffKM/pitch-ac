# pitch-ac PRD v2 — Man City 카툰 팬 사이트

## 핵심 정보

**목적**: 맨체스터 시티 팬이 경기 데이터를 카툰 캐릭터와 함께 체험하고, 카드를 생성하여 공유하는 팬 사이트.
**사용자**: 맨체스터 시티 팬 — 한국 사용자 우선 타깃, 이후 글로벌 확장.
**피벗 배경**: PL 전체 데이터 플랫폼에서 맨시티 단일 팀 카툰 팬 사이트로 전환. 기존 SportMonks API, Supabase DB, TanStack Query 폴링, satori OG 이미지 인프라를 활용하되, Lottie 애니메이션과 Supabase Storage 기반 카툰 에셋 시스템을 추가한다.

---

## 사용자 여정

```
1. 매치데이 대시보드 (진입점)
   ↓ 앱 첫 접근 시 자동 리디렉션 (현재 게임위크 맨시티 경기)

2. 맨시티 경기 포커스
   ↓ 경기 카드 클릭

3. 경기 상세 — 카툰 피치 뷰
   ↓ 라이브 경기: 카툰 캐릭터 감정 변화 + 말풍선 이벤트
   ↓ 골 발생 시: 이벤트 리액션 오버레이 + 무드 리액션 참여
   ↓ 경기 종료 시: 포스트매치 카툰 카드 자동 생성

4. 포스트매치 카드 공유
   ↓ "Share" 버튼 → Web Share API 또는 이미지 다운로드

5. 갤러리에서 과거 카드 탐색
   ↓ 갤러리 → 카드 클릭 → 공유

사이드 여정:
A. 스쿼드 → 카툰 아바타 그리드 → 선수 클릭 → 카툰 프로필
B. 비교 → 맨시티 선수 간 카툰 배틀카드 생성 → 공유
```

### 신규 테이블

#### cartoon_assets (카툰 에셋)

| 필드       | 설명                                          | 타입/관계    |
| ---------- | --------------------------------------------- | ------------ |
| id         | 고유 식별자                                   | UUID         |
| player_id  | 선수                                          | → players.id |
| mood       | 감정 상태 (default/happy/angry/sad/celebrate) | Text         |
| image_url  | Supabase Storage 이미지 URL                   | Text         |
| lottie_url | Lottie JSON 애니메이션 URL (nullable)         | Text         |
| asset_type | 에셋 유형 (avatar/full/pitch)                 | Text         |
| created_at | 생성 시각                                     | Timestamp    |
| updated_at | 마지막 갱신 시각                              | Timestamp    |

> - `mood`: 감정별 카툰 이미지. `default`는 기본 포즈.
> - `asset_type`: `avatar`(썸네일 64x64), `full`(프로필 대형 400x600), `pitch`(포메이션 뷰 128x128).
> - `lottie_url`: idle/celebration 등 Lottie 애니메이션 JSON 파일 경로. 정적 이미지만 있는 경우 null.
> - 복합 유니크 제약: `(player_id, mood, asset_type)`.

#### speech_bubbles (말풍선 대사)

| 필드       | 설명                                          | 타입/관계    |
| ---------- | --------------------------------------------- | ------------ |
| id         | 고유 식별자                                   | UUID         |
| player_id  | 선수 (nullable, null이면 범용 대사)           | → players.id |
| trigger    | 트리거 이벤트 (goal/assist/card/substitution) | Text         |
| text       | 말풍선 대사 텍스트                            | Text         |
| locale     | 언어 코드 (ko/en)                             | Text         |
| created_at | 생성 시각                                     | Timestamp    |

> - `player_id`가 null이면 해당 trigger에 대한 범용 대사(어떤 선수에게든 적용 가능).
> - `player_id`가 지정되면 해당 선수 고유 대사 우선 사용, 없으면 범용 폴백.
> - `locale`: 한국어 우선, 향후 다국어 확장 대비.

#### gallery_cards (갤러리 카드)

| 필드       | 설명                                      | 타입/관계     |
| ---------- | ----------------------------------------- | ------------- |
| id         | 고유 식별자                               | UUID          |
| fixture_id | 경기                                      | → fixtures.id |
| card_type  | 카드 유형 (mom/result_summary/highlight)  | Text          |
| image_url  | 생성된 카드 이미지 URL (Supabase Storage) | Text          |
| title      | 카드 제목 (예: "Haaland MOM - GW32")      | Text          |
| metadata   | 추가 데이터 (선수 ID, 스탯 등)            | JSONB         |
| created_at | 생성 시각                                 | Timestamp     |

> - `card_type`: `mom`(최고 평점 선수), `result_summary`(경기 결과 요약), `highlight`(명장면/골).
> - 경기 종료 시 `lib/services/gallery/card-generator.ts`에서 자동 생성.
> - `metadata` 예시: `{ "player_id": 123, "goals": 2, "rating": 9.1 }`.

#### mood_reactions (무드 리액션 집계)

| 필드       | 설명                                      | 타입/관계     |
| ---------- | ----------------------------------------- | ------------- |
| id         | 고유 식별자                               | UUID          |
| fixture_id | 경기                                      | → fixtures.id |
| user_id    | 사용자 (로그인 사용자만)                  | → auth.users  |
| mood       | 선택한 감정 (happy/angry/tense/celebrate) | Text          |
| created_at | 생성 시각                                 | Timestamp     |

> - 라이브 경기 중 사용자별 감정 투표 기록.
> - 동일 fixture에 대해 사용자당 1분에 1회 제한 (Rate Limit).
> - 집계 쿼리로 현재 감정 분포 퍼센티지 계산: `SELECT mood, COUNT(*) FROM mood_reactions WHERE fixture_id = X GROUP BY mood`.
